# Workflows

This document describes representative workflows. It intentionally avoids private schedules, chat IDs, credentials, and production configuration.

## 1. Morning AI Governance Brief

Purpose: produce a concise overview of relevant AI governance, AI policy, institutional, and technical developments.

Typical steps:

1. Scan trusted sources, recent news, institutional updates, and research signals.
2. Filter for relevance to AI governance, diplomacy, and public-interest projects.
3. Deduplicate against previous briefings.
4. Summarize developments with implications.
5. Flag items that require human attention.

Human role: read, decide what matters, and request follow-up.

## 2. AI Research Radar

Purpose: track technical developments that may affect governance, policy, or project strategy.

Typical steps:

1. Monitor model releases, technical papers, talks, benchmarks, and developer ecosystem shifts.
2. Explain why a development matters.
3. Translate technical changes into governance or product implications.
4. Add durable concepts to the second brain where useful.

## 3. Project Health Check

Purpose: keep public-interest tools from silently decaying.

Typical steps:

1. Check deployment and runtime status.
2. Inspect recent errors or failed scheduled tasks.
3. Run tests or lightweight verification when appropriate.
4. Summarize risks and recommended fixes.
5. Ask for approval before risky, external, or irreversible action.

## 4. Diplomatic Analysis Drafting

Purpose: assist with structured analysis while preserving human authorship and judgment.

Typical steps:

1. Retrieve relevant context from memory and the second brain.
2. Identify stakeholders, positions, incentives, and institutional constraints.
3. Draft a structured memo or argument map.
4. Stress-test assumptions and counterarguments.
5. Leave final judgment and external use to the human operator.

## 5. Knowledge Graph Maintenance

Purpose: convert raw activity into durable knowledge.

Typical steps:

1. Review recent daily memory logs and project notes.
2. Identify durable entities, concepts, and projects.
3. Update Markdown wiki pages.
4. Add wikilinks between related concepts.
5. Keep the vault useful for both human review and agent retrieval.
